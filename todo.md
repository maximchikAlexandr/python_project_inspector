# План развития отчета по связанности модулей

## Что нужно добавить в граф

### 1. Режим центрирования по одному модулю

Добавить возможность:
- выбрать один модуль;
- центрировать граф по нему;
- просматривать его связи на заданном уровне глубины.

### 2. Режим просмотра всех модулей

Сохранить отдельный режим, в котором отображаются все модули без центрирования на одном выбранном модуле.

## Что нужно добавить в настройки графа

Добавить настраиваемые параметры для управления отображением графа:
- увеличить силу притяжения между нодами;
- уменьшить силу притяжения между нодами;
- настроить точность;
- настроить толщину линий связей;
- настроить расстояние связей.

## Что нужно доработать в UI

### 1. Более удобное зуммирование

Доработать зуммирование графа, чтобы им было удобнее пользоваться.

### 2. Фильтрация и настройка окраски модулей

Добавить фильтрацию, связанную с окраской модулей.

Окраска модулей должна учитывать показатели, влияющие на яркость модуля.

Нужно сделать так, чтобы можно было отдельно включать каждый показатель, влияющий на окраску, например:
- отдельные показатели сложности;
- количество строк кода.

## Что нужно добавить со стороны Python

### 1. Поддержка большего количества модулей

Добавить возможность включать в анализ больше модулей.

### 2. Совместный анализ проекта и Core Code

Добавить возможность анализировать вместе:
- целевой проект;
- Core Code.

### 3. Разные цвета для модулей из разных источников

Сделать так, чтобы:
- Core Addon из Core Code;
- Addon из целевого проекта

подсвечивались разными цветами.
